## Bugs Discovered & Fixed During E2E

### BUG-E2E-001: Router hash query string parsing failure
**Severity**: Critical вЂ” breaks all filter/search functionality
**Root cause**: `router.js` used `window.location.hash.slice(1)` directly to compute `root`. When filters applied query params (e.g. `/#/explore?q=е†°еІ›`), `root` became `/explore?q=е†°еІ›`, which didn't match any route key. Router fell back to `routes['/']` (Home page), silently replacing the Explore page.
**Fix**: Extract path before `?` before splitting:
```js
const path   = hash.split('?')[0];
const parts  = path.split('/').filter(Boolean);
const root   = '/' + (parts[0] || '');
const handler = routes[path] || routes[root] || routes['/'];
```
**Files**: `web/js/router.js`

### BUG-E2E-002: Detail page API envelope unwrapping
**Severity**: High вЂ” detail page renders empty
**Root cause**: Backend `GET /api/journeys/:slug` returns `{ data: journey }`. `detail.js` passed the entire envelope to `_renderPage()`, so `data.title` was undefined and all fields were empty.
**Fix**: Unwrap `.data` before passing to renderer:
```js
API.getJourney(slug)
  .then((res) => this._renderPage(res.data || res))
```
**Files**: `web/js/pages/detail.js`

### BUG-E2E-003: `const Pages` redeclaration across script tags
**Severity**: High вЂ” total JS failure in Chromium
**Root cause**: Multiple `<script>` tags with top-level `const Pages = window.Pages || {}` share lexical scope in Chromium. Second script throws `SyntaxError: Identifier 'Pages' has already been declared`.
**Fix**: Changed to `var Pages = window.Pages || {}` in all page files.
**Files**: `web/js/pages/home.js`, `explore.js`, `detail.js`

### BUG-E2E-004: Router `this` context loss
**Severity**: High вЂ” page methods fail when routed
**Root cause**: `Router.define('/', Pages.Home.render)` stored unbound method reference. When called by router, `this` was `undefined`, causing `this._buildHero is not a function`.
**Fix**: Added `ctx` parameter to `define()` and bound handler:
```js
function define(path, handler, ctx) {
  routes[path] = ctx ? handler.bind(ctx) : handler;
}
Router.define('/', Pages.Home.render, Pages.Home);
```
**Files**: `web/js/router.js`

### BUG-E2E-005: Seed data tag slug mismatch
**Severity**: Medium вЂ” seed associations broken
**Root cause**: `db/seed.sql` `journey_tags` INSERTs used Chinese tag names (`'е­¤з‹¬ж„џ'`) in `WHERE t.slug IN (...)` but actual slugs are English (`'solitude'`).
**Fix**: Changed all 5 INSERT statements to use English slugs.
**Files**: `db/seed.sql`
